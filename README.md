34, 35, 36 — это "input-only" пины. У них нет встроенных подтяжек (pull-up/pull-down), поэтому ваши внешние резисторы на 1 кОм к 3.3В обязательны и правильны.

Логика сигналов: Если энкодер замыкает на "массу" (GND), то при разомкнутом состоянии на пине будет 1 (высокий уровень), а при замыкании — 0 (низкий).

Резисторы 1 кОм: Это довольно сильная подтяжка (ток ~3.3мА). Если энкодер потребляет слишком много или есть помехи, это хорошее решение, но обычно хватает и 4.7к–10к.

Фильтрация: В коде выше включен glitch_filter. Для "шумных" механических энкодеров это критично, чтобы не было ложных срабатываний.

Поскольку у вас появился сигнал Z (Index) на 36-м пине, в ESP-IDF 5.5 его лучше всего использовать для автоматического сброса счетчика в ноль при каждом полном обороте.

Логика Z: Сигнал Z у этой модели очень короткий (один импульс на оборот). На высокой скорости ESP32 может «проспать» его через обычный GPIO. Если вам нужна идеальная точность на больших оборотах, сигнал Z можно завести на второй юнит PCNT как управляющий сигнал или использовать pcnt_unit_add_watch_point.

Разрешение: Помните, что PCNT в режиме x4 выдаст в 4 раза больше импульсов, чем написано на корпусе энкодера (например, если на энкодере 1000 P/R, в счетчике будет 4000 на оборот).

Аппаратный Z-сброс (Index):
В токарном станке фаза Z критична для нарезания резьбы (синхронизация начала прохода). Чтобы ESP32 автоматически обнуляла счетчик PCNT по сигналу Z без участия процессора, используйте настройку pcnt_unit_set_watch_point. Но так как в ESP32 PCNT нет прямого аппаратного входа "Reset", обычно это делают через прерывание с минимальной задержкой.

Высокая помехозащищенность:
Станок — источник сильных электромагнитных помех (двигатель, частотник).
- Ваши 1 кОм подтяжки — это отлично (лучше, чем стандартные 10к).

Использование IRAM:
Все функции обработки энкодера и прерывания Z-метки должны быть помечены атрибутом IRAM_ATTR. Это исключит задержки, связанные с чтением из Flash-памяти.

Гибкость настроек.
Если вы смените энкодер (например, с 600 импульсов на 5000 на оборот), вам придется уменьшать значение фильтра, чтобы не терять шаги.


```kconfig
config PCNT_GLITCH_FILTER_NS
    int "Glitch filter time (nanoseconds)"
    range 0 12500
    default 12500
    help
        Игнорировать импульсы короче этого времени.
        12500 нс (12.5 мкс) подходит для медленных энкодеров.
        Для скоростных (высокое разрешение или обороты) уменьшите до 1000-5000.
```

Теперь в коде вместо жесткого числа используйте макрос:
```c
// В функции init_encoder_module:
pcnt_glitch_filter_config_t filter_config = {
    .max_glitch_ns = CONFIG_PCNT_GLITCH_FILTER_NS
};
ESP_ERROR_CHECK(pcnt_unit_set_glitch_filter(*unit, &filter_config));
```

Совет по выбору значения:
- 12500 (макс): Максимальная защита. Потолок частоты — около 40 кГц.
- 1000: Для очень быстрых систем. Позволяет работать на частотах до 500 кГц.
- Для ручного энкодера: Оставляйте максимум (12500), это лучшее спасение от дребезга.

Чтобы энкодер работал стабильно и не пропускал шаги, фильтр должен быть короче, чем самая узкая фаза сигнала при максимальной скорости.

1. Расчет частоты и времени импульса
При режиме X4 на каждый физический цикл (период) энкодера приходится 4 фронта. Нам нужно рассчитать длительность одного "полупериода" (состояния), так как именно его фильтр может случайно подавить.

- Максимальные обороты в секунду:
2500 об/мин / 60 = 41,67 об/сек

- Частота импульсов (линий) в секунду:
41,67 * 1800 линий = 75000Гц (75кГц)

- Период одной линии:
1 / 75000 = 13,33 мкс

- Длительность одной фазы (состояния):
В идеальном энкодере фаза длится 1/4 периода (из-за сдвига A и B на 90°).
13,33 мкс / 4 = 3333 нс

2. Подбор значения для фильтра
Фильтр должен быть меньше длительности фазы, иначе он примет полезный сигнал за помеху. Обычно берут запас в 30-50%.

- Ваше расчетное время фазы: 3333 нс.
- Рекомендуемое значение фильтра: 2000–2500 нс.

С учетом ваших данных, диапазон в Kconfig стоит подправить, чтобы не ввести случайно слишком большое значение:

```kconfig
config PCNT_GLITCH_FILTER_NS
    int "Glitch filter time (nanoseconds)"
    range 0 12500
    default 2500
    help
        Для 1800 линий на 2500 об/мин теоретический предел ~3300 нс.
        Значение 2500 нс обеспечит надежную фильтрацию и работу на макс. скорости.
```

Для вашего проекта на 2500 об/мин установите в menuconfig значение 2500. Это защитит от высокочастотных наводок электроники, но позволит PCNT видеть каждый из 7200 тиков на оборот.
